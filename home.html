Redesign Project

Introduction

For my redesign project, I chose to focus on Rhode Island’s unemployment insurance portal, known as the “internet claims system”. This system is used by thousands of Rhode Islanders to claim essential unemployment benefits, and has seen increasing use since the COVID-19 pandemic as many continue to struggle without work.

Like many state unemployment portals, this system is outdated and poorly maintained, even using backend technology from the 1950s to process claims. However, we will be looking at the frontend of the system, namely the form that Rhode Islanders need to fill out to claim benifits.

It is essential to vulnerable Rhode Islanders that this portal function seamlessly. However, there are a number of glaring issues with the form pages, including:

  • Use of ASP.NET Web Forms, which has been replaced by newer standards
  • Poor scaling, resulting in difficulty completing the form on some devices
  • No privacy when entering SSN and other sensitive details
  • Outdated browser compatibility, recommending use of IE 5.0 or Firefox 2.0
  • Generally poor visual design language

I also identified the following accessibility issues:

  • Low contrast, including blue-on-blue text, which is difficult to read for some people
  • No fieldsets for questions, making it difficult for screen readers to access the forms
  • Use of images to denote asterisks, with no alt-text
  • No page regions or headings
  • No language header

When envisioning a redesigned unemployment portal, I came up with a list of objectives:

  • Scalable for use on any device
  • Clean and free of visual clutter and confusing design
  • Modern, making use of new web technologies
  • Accessible to users of a screen reader or translation software

As part of achieving these objectives, I decided to use Boostrap, a popular open-source toolkit for building responsive websites. Bootstrap provides ready-made styling for common page elements, as well as automatic scaling. Bootstrap's modern style should be familiar to most internet users:

Technical and Accessibility Upgrades

Since accessibility and use of modern technical standards were priorities for me, I implemented the following features on my redesign:

  • Lightweight page for better performance
  • HTML5 form validation, replacing images of asterisks
  • Modern input boxes with input format requirements
  • ARIA descriptions for form inputs with subtext
  • Fieldsets for screen readers
  • Language header
  • Page regions and headings
  • Higher-contrast design
